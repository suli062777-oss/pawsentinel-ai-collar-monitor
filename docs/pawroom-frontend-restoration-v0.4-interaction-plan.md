# PawRoom 前端还原 v0.4 交互联动计划

日期：2026-07-08  
目标：在不改变 v0.3 页面结构和视觉方向的前提下，让桌面小世界、今日历程、轻互动共享同一组事件状态，提升“真实产品”感。

## 1. 不改变的内容

- 不新增一级入口。
- 不新增顶部栏或右侧抽屉。
- 不把桌面小世界改成通用数据后台。
- 不改变首页、宠物数据、今日历程的页面结构。
- 不引入第三方动画库，继续使用原生 CSS/JS，避免依赖不确定。

## 2. 新增状态模型

把今日事件抽象为 `worldRoute`：

- 时间
- 区域
- 状态文案
- 宠物坐标
- 对应行为类型
- 来源

该模型同时驱动：

- 桌面小世界宠物位置。
- 桌面小世界行为浮层高亮。
- 小世界路径节点高亮。
- 今日历程时间线当前态。
- 播放路径时的状态推进。

## 3. 交互规则

### 3.1 路径播放

- 点击“播放路径”后，从第一个事件开始按顺序推进。
- 每一步更新 `activeJourneyEvent`。
- 回放到最后一个事件后自动停止，不无限循环。
- 动效只使用 `translate` 和 `opacity`，避免布局抖动。

### 3.2 小世界节点

- 小世界场景内显示轻量路径节点。
- 当前节点高亮，历史节点弱高亮，未来节点保持低透明。
- 点击节点可跳转到该事件。

### 3.3 轻互动

- 点击“投喂”会跳到饭盆相关状态。
- 点击“扔玩具”会跳到玩具/客厅活动状态。
- 点击“叫名字”会跳到等主人/回应状态。
- 点击“摸摸”只改变当前互动反馈，不强制改变路径。

## 4. 动效边界

- 宠物移动：`translate` 过渡，600ms 左右。
- 当前节点：轻微呼吸，不闪烁。
- 行为卡：当前态轻微橙色高亮。
- 回放结束：toast 反馈。
- 尊重 `prefers-reduced-motion`，关闭移动和循环动画。

## 5. 验收

- 8 个页面仍可访问。
- 桌面小世界和今日历程都能点击路径节点。
- 今日历程播放会推进当前事件。
- 小世界按钮点击有反馈，并能影响相关场景状态。
- 不出现乱码、emoji、顶部栏、右侧抽屉。

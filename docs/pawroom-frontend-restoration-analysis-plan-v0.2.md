# PawRoom 前端高保真还原分析与执行计划 v0.2

## 目标

将现有 PawRoom 高保真页面图片还原为可交互前端页面。优先最大化还原设计稿的结构、比例、层级、留白、圆角、配色和视觉氛围，而不是重新优化设计。

## 还原原则

1. 先拆解页面，再写代码。
2. 不自行改变布局、风格、配色、圆角和留白。
3. 不擅自增加设计稿中没有的模块。
4. 不把页面改成通用后台 dashboard。
5. 优先还原结构、比例、层级、留白和视觉氛围。
6. 资源缺失时可以占位，但位置、尺寸和层级必须一致。
7. 未来要接真实功能的部分必须组件化、数据化，不能写死业务逻辑。
8. 宠物、房间、地图、漫画等内容素材可以像素化；UI 控件、文字、图表、卡片继续保持现代液态玻璃风格。
9. 图片中的文字必须用真实 DOM 文本重建，不能依赖模糊截图文字。
10. 每次大更新必须有审查和二次修正阶段。

## 页面结构拆解

### 全局 App Shell

- 固定 `1440 x 1024` 设计舞台。
- 左侧为唯一全局任务栏。
- 左侧栏上半部分：首页、桌面小世界、宠物数据、今日历程、设置。
- 左侧栏下半部分：添加、电量、宠物信息。
- 不设置顶部全局栏。
- 不设置右侧固定抽屉栏。
- 主内容区从左侧栏右侧开始，页面切换只替换主内容。

### 首页

- 上半部分为橙色液态玻璃主视觉板块。
- 主视觉包含当前状态、大号关键数值、睡眠宠物像素素材。
- 中部保留轻量时间线或状态路径连接。
- 下半部分为错位数据卡片：活动量、休息时长、心率趋势、呼吸趋势、提醒、数据置信度。
- 不回退到地图型首页。

### 桌面小世界

- 主体为沉浸式 2D 像素房间场景。
- 场景上叠加：状态卡、安全等级、当前区域、项圈连接、电量、小窗预览。
- 场景内包含安全区虚线、活动路径、区域标签、宠物分身。
- 互动按钮靠近场景底部：摸摸、投喂、叫名字、扔玩具。
- 底部或角落只保留轻量数据卡，不喧宾夺主。

### 宠物数据

- 顶部状态行：项圈状态、电量、最近同步、数据置信度、当前区域。
- 中部为数据仪表：活动量、休息时长、心率、呼吸、区域停留、活动节律。
- 小地图只是辅助数据卡，不作为全页主体。
- 底部保留数据边界：项圈记录、用户补充、AI 演绎、医疗边界说明。

### 今日历程

- 顶部为今日小剧场/今日摘要主卡。
- 中部为今日时间线和路径回放。
- 创作区包括 AI 小剧场、四格漫画、表情包、角色卡、分享导出。
- Credits 消耗必须在生成前提示。
- 它是故事编辑台，不是普通后台时间线。

### 设置

- 只保留功能型分组：安全区域、提醒偏好、Paw Credits、数据隐私、医疗边界。
- 使用液态玻璃/实底卡片分组，不需要强主视觉。
- 开关、选择项和说明卡必须保持简洁。

### 添加宠物

- 使用渐进式 Stepper，降低一次性填写负担。
- Step 1：上传照片或选择示例宠物。
- Step 2：填写必要基础信息。
- Step 3：选择像素视觉风格。
- Step 4：生成宠物电子分身。
- Step 5：连接项圈数据。
- 每一步只保留一个主要动作和一个预览区。

### 电量

- 主卡展示电量百分比、项圈在线状态、预计可用时长、最近同步。
- 辅助卡展示低电量提醒、同步频率、充电建议和设备信息。
- 低电量、离线状态要同步影响首页、小世界和数据页。

### 宠物信息

- 主体包含像素头像、昵称、品种、年龄、体重、性格标签。
- 素材库包含照片、声音、表情包、今日事件、角色卡、小剧场。
- 设备区展示当前绑定项圈和同步状态。

## 布局关系

- 设计坐标以 `1440 x 1024` 为唯一基准。
- 源图 `1487 x 1058` 需要缩放到舞台基准后再取坐标。
- 左侧栏宽度保持稳定，不随内容撑开。
- 主内容区不同页面可使用绝对定位 + CSS Grid 混合方式还原设计稿。
- 场景页使用背景素材/房间资产 + DOM 浮层。
- 数据页使用真实 DOM 卡片和 SVG 图表，避免整图贴图。

## 字体层级

- 字体栈：`-apple-system`, `SF Pro Display`, `SF Pro Text`, `PingFang SC`, `Microsoft YaHei`, `Noto Sans SC`, sans-serif。
- Display：48-56px，用于首页主状态。
- Large Number：72-138px，用于关键数值。
- Title LG：24-34px，用于页面和模块标题。
- Title MD：18-22px，用于卡片标题。
- Body：14-17px，用于说明文字。
- Caption：12-13px，用于时间、来源、辅助说明。
- 数字统一使用 `font-variant-numeric: tabular-nums`。

## 色彩层级

- 主色：`#FF8F3F`。
- 深橙：`#F76F23`，用于主按钮和路径强调。
- 奶油底：`#FAF8F3` / `#F6F1E8`。
- 主文字：`#2B2521`。
- 正文：`#4D4741`。
- 辅助文字：`#8D8B73`。
- 描边：`#D9D2C8` 或半透明白。
- 安全：`#68AB12`。
- 注意：暖琥珀/橙色。
- 异常：`#E2574C`，只用于真正异常或需关注状态。

## 需要精确还原的关键点

1. 左侧栏上 5 + 下 3 的分组关系。
2. 无顶部全局栏、无右侧固定抽屉。
3. 所有页面保持 `1440 x 1024` 舞台比例。
4. 首页保留橙色大主视觉 + 睡眠宠物 + 下方数据卡结构。
5. 桌面小世界保留沉浸式像素房间，不改成数据后台。
6. 宠物数据页保持数据仪表逻辑，不套用首页布局。
7. 今日历程保持时间线 + 回放 + AI 创作输出。
8. 文案使用真实中文 DOM，不依赖图片文字。
9. 图标使用 SVG，不使用结构性 emoji。
10. 医疗边界说明必须保留。

## 还原风险点

| 风险 | 影响 | 处理 |
| --- | --- | --- |
| 源图文件名无语义 | 页面对应可能错误 | 建立映射表并人工复核 |
| 早期文档乱码 | 交接和审查困难 | 使用 v0.2 文档作为后续准绳 |
| 源图尺寸不是 1440x1024 | 坐标偏移 | 统一缩放到舞台坐标 |
| 场景细节过复杂 | CSS 难以还原 | 场景和道具拆为 image2 透明资产 |
| AI 图中文字模糊 | 视觉不专业 | 所有文字用 DOM 重建 |
| 数据页后台化 | 偏离 C 端陪伴产品 | 保持错位、玻璃、像素资产的氛围 |
| 动效过多 | 干扰看护主线 | 动效只服务状态变化和轻互动 |
| 低电量/离线状态缺失 | 产品不可信 | 已在 v0.5 加入状态分支 |

## 分阶段执行计划

| 阶段 | 任务 | 输出 | 审查点 |
| --- | --- | --- | --- |
| P0 | 主框架资源映射 | 图片到路由映射表 | 页面不拿错 |
| P1 | AppShell 还原 | 固定舞台和左侧栏 | 无顶部栏、无右抽屉 |
| P2 | 8 页静态结构 | 主要页面 DOM/CSS | 结构和比例优先 |
| P3 | 素材拆分 | image2 透明素材清单 | 不用整图贴片 |
| P4 | 数据组件化 | Mock 数据和组件边界 | 后续可接真实数据 |
| P5 | 状态分支 | 正常/低电量/离线/需关注 | 状态影响多页面 |
| P6 | 交互动效 | 路径回放、按钮反馈、宠物低频动效 | 不改变布局 |
| P7 | 截图审查 | 1440x1024 截图和总览图 | 查乱码、溢出、错位 |
| P8 | 二次修正 | 修正后的页面 | 还原优先，不擅自优化 |

## 下一轮建议

优先做 `桌面小世界` 的资产层精修：它是产品差异化最强的页面，也是当前 CSS 绘制和高保真源图差距最大的页面。下一轮应先把房间背景、宠物分身、饭盆、玩具、宠物窝、沙发等拆成 image2 透明素材需求，再决定哪些先用占位、哪些替换为图片资产。

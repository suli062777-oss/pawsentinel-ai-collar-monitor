# PawRoom 宠物项圈硬件产品设计 v0.1

版本：v0.1  
日期：2026-07-08  
定位：概念验证 / 产品汇报用硬件方案，不作为已量产规格承诺

## 1. 硬件产品定位

PawRoom Collar 是 PawRoom 平台的数据入口。它首先解决宠物独自在家或外出时的安全焦虑：知道宠物在哪里、是否离开安全区域、活动是否异常、生命状态趋势是否需要关注。软件侧再把这些冷数据转译成桌面小世界、今日回放和低干扰提醒。

一句话：

> PawRoom Collar 用定位、活动与生命状态趋势数据提供宠物安全看护，再把这些数据交给 PawRoom 软件生成可陪伴、可互动、可分享的宠物小世界。

不建议把它定义成“医疗项圈”或“精准室内定位项圈”。更稳的表达是：

- 安全看护项圈
- 活动路径与状态趋势采集器
- 非医疗生命状态趋势提醒
- PawRoom 桌宠小世界的数据底座

## 2. 竞品与机会边界

现有强竞品已经覆盖了定位、活动、睡眠、健康趋势等硬件能力：

- Tractive：GPS 定位、虚拟围栏、位置历史、活动/睡眠、健康提醒。
- Fi：GPS、活动、睡眠，以及部分行为变化识别。
- PetPace：偏健康监测项圈，覆盖体温、脉搏、呼吸等生命体征方向。
- Invoxia / FitBark / Halo 等：分别在健康趋势、活动、GPS 围栏等方向有成熟产品。

所以 PawRoom Collar 不应正面比拼“最强 GPS”“最专业医疗监测”“最强围栏训练”。它的差异应该是：

1. **硬件可靠收集基础安全数据。**
2. **软件把数据变成低干扰、好理解、好看的状态表达。**
3. **PawRoom 桌宠和今日小剧场提升日常打开频次。**

## 3. 推荐硬件架构

### 3.1 总体结构

推荐采用“外侧主控舱 + 内侧柔性传感窗 + 可拆卸项圈带”的结构。

| 部件 | 位置 | 作用 |
| --- | --- | --- |
| 外侧主控舱 | 项圈外侧上方或侧前方 | 放置 GNSS、蜂窝通信、BLE、主控、电池、天线、充电触点 |
| 内侧柔性传感窗 | 贴近颈部皮肤一侧 | 放置 PPG 光学窗、皮温传感、接触检测、柔性软垫 |
| 项圈带 | 围绕颈部 | 佩戴固定、抗拉、亲肤、防水、可清洁 |
| 快拆扣 | 侧边 | 防勒、防卡、方便充电和清洁 |
| 磁吸充电触点 | 主控舱底部或侧面 | 防水充电，不开孔 |

为什么不用“单纯一个挂件”：

- GPS / LTE 天线希望在外侧，尽量无遮挡。
- 心率 / 呼吸 / 皮温趋势需要更稳定的皮肤接触。
- 电池、天线、传感器全部塞在一个吊牌里，会更晃动，生命趋势数据更不稳定。

### 3.2 核心能力分层

| 能力 | 建议实现 | 可承诺程度 |
| --- | --- | --- |
| 户外定位 | GNSS + LTE-M / NB-IoT / 4G Cat.1 数据回传 | 可作为核心能力 |
| 室内大概区域 | BLE / Wi-Fi 指纹 / 家庭信标 + IMU 推断 | 可做“区域状态”，不承诺厘米级定位 |
| 安全围栏 | GPS 地理围栏 + 家庭区域围栏 | 可作为安全提醒 |
| 活动状态 | 6 轴 IMU 识别静止、走动、活跃、异常高活动 | 可作为核心能力 |
| 睡眠/静息 | IMU + 时间段 + 低活动模式推断 | 可作为趋势能力 |
| 心率趋势 | PPG 光学传感，优先在静息、接触良好时采样 | 只承诺趋势，不承诺医疗级准确 |
| 呼吸趋势 | 静息时由微运动 / PPG / IMU 信号推断 | 只承诺趋势提醒 |
| 皮温趋势 | 内侧温度传感器，做相对变化 | 只做趋势，不等于体温 |
| 叫声/异常声音 | 可选麦克风，隐私和功耗需谨慎 | 建议后置，不作为 P0 |

## 4. 传感与通信模块

### 4.1 主控与通信

推荐两条路线：

| 路线 | 适合阶段 | 说明 |
| --- | --- | --- |
| 蜂窝 SoC 一体方案 | 概念样机 / 高集成产品 | 如 Nordic nRF9160 类 LTE-M/NB-IoT + GNSS 方案，集成度高，适合小型穿戴设备 |
| MCU + 蜂窝/GNSS 模块 | 工程验证 / 供应链灵活 | 主控 MCU 管理传感器，独立 GNSS/蜂窝模块负责定位与回传 |

推荐 P1 概念规格：

- GNSS：GPS / BeiDou / Galileo / GLONASS 多星座定位。
- 蜂窝：LTE-M / NB-IoT 优先；如果目标市场网络支持不足，可考虑 Cat.1。
- BLE：手机配网、近场同步、家庭信标识别。
- Wi-Fi 扫描：辅助判断在家/离家，不建议持续联网。

参考来源：

- Nordic nRF9160 产品页：https://www.nordicsemi.com/Products/nRF9160
- u-blox GNSS 模块参考：https://www.u-blox.com/en/positioning-chips-and-modules

### 4.2 运动与生命状态

推荐 P1 传感组合：

| 传感器 | 作用 | 注意点 |
| --- | --- | --- |
| 6 轴 IMU | 活动、步态、静止、睡眠、异常高活动 | 最稳、最应该作为 P0 |
| PPG 光学传感 | 心率趋势、静息状态下的脉搏变化 | 宠物毛发、运动和接触压力会影响准确性 |
| 皮温传感 | 贴肤温度趋势 | 不是核心体温，不能医疗化表述 |
| 接触检测 | 判断项圈是否佩戴、传感窗是否贴合 | 提升数据可信度 |
| 气压/环境温湿度 | 可选，用于环境上下文 | 不是 P0 |

建议在软件里为每个生命趋势数据加“数据置信度”：

- 接触良好 + 静息状态：可显示心率/呼吸趋势。
- 运动中 + 接触不稳：降级为活动状态，不显示精确趋势。
- 佩戴异常：提醒用户调整项圈松紧。

参考来源：

- Bosch BMI270 IMU 参考：https://www.bosch-sensortec.com/products/motion-sensors/imus/bmi270/
- Analog Devices MAX30101 PPG 参考：https://www.analog.com/en/products/max30101.html

## 5. 数据采集策略

### 5.1 正常模式

- 活动状态：每 30-60 秒汇总一次。
- 室内区域：每 1-5 分钟更新一次。
- 心率/呼吸/皮温：优先在静息且接触良好时采样。
- 手机/桌面端：展示“当前状态 + 今日趋势”，不制造过度提醒。

### 5.2 离家 / 户外模式

- GPS 低频更新：1-5 分钟一次。
- 外出散步轨迹：按行程记录。
- 离开安全围栏：立即提升定位频率并推送提醒。

### 5.3 丢失模式

- GPS 高频更新：约 5-15 秒一次。
- 开启蜂窝持续回传。
- 可开启蜂鸣 / LED 闪烁，但默认不惊吓宠物。
- 丢失模式会显著耗电，需在界面明确提示。

## 6. 续航与电源

建议目标：

| 模式 | 目标续航 |
| --- | --- |
| 家庭日常看护 | 5-7 天 |
| 外出散步记录 | 2-4 天 |
| 丢失高频定位 | 8-24 小时 |

电源设计建议：

- 250-500 mAh 锂电池，根据猫/小型犬/中大型犬分型号。
- 磁吸触点充电，避免开孔。
- 低电量 20% / 10% 两级提醒。
- 软件端显示“预计还能使用多久”，比单纯百分比更易理解。

## 7. 外观与佩戴设计

### 7.1 设计关键词

- 温暖
- 简洁
- 亲肤
- 高级但不冷
- 像宠物日用品，不像监控设备
- 有 PawRoom 的可爱感，但不幼稚

### 7.2 形态建议

推荐形态：**柔和胶囊主控舱 + 圆角项圈带 + 内侧软垫传感窗**。

设计细节：

- 主控舱：鹅卵石/胶囊形，边缘大圆角。
- 表面：哑光微磨砂，不做高反光塑料。
- Logo：小型爪印或极简 PawRoom 标记，低调压印。
- 指示灯：一颗柔和点状 LED，不做强科技灯带。
- 传感窗：内侧半透明暖灰/琥珀色小窗。
- 扣具：隐藏式磁吸扣或低调快拆扣。
- 防水：目标 IP67，表达为“日常防水、防雨、可擦洗”。

### 7.3 三套配色

| 配色 | 主色 | 辅助 | 气质 |
| --- | --- | --- | --- |
| 深棕色 | #4A2A1A | 暖铜、深咖、米白 | 高级、耐脏、像皮具 |
| 暖白色 | #F4E8D7 | #FF8F3F、浅沙色 | 温暖、柔软、亲近 |
| 纯白色 | #FFFFFF | 银灰、浅橙、浅雾蓝 | 科技、干净、轻量 |

## 8. 建议 P0 / P1 / P2

### P0：必须讲清楚

- GPS / GNSS 户外定位。
- 安全围栏和离开提醒。
- 活动状态识别。
- 静息/睡眠趋势。
- 心率/呼吸/皮温趋势的非医疗提醒。
- 电量、佩戴状态、连接状态。
- 数据置信度。

### P1：增强可信度

- 家庭区域识别：客厅、门口、饭盆、宠物窝等。
- 丢失模式。
- 低电量预测。
- 异常静止 / 长时间门口徘徊 / 活动突增提醒。
- 可拆洗项圈带。

### P2：后续探索

- UWB 室内定位套装。
- 麦克风识别叫声。
- 多宠家庭协同。
- 兽医报告导出。
- 宠物保险 / 健康服务联动。

## 9. 风险与表述边界

需要避免：

- “诊断疾病”
- “精准心率”
- “精准体温”
- “实时监视宠物全部行为”
- “GPS 室内精确定位”

建议表述：

- “生命状态趋势”
- “异常趋势提醒”
- “看护参考”
- “非医疗用途”
- “基于项圈数据和 AI 的状态理解”
- “室内大概区域”

## 10. 推荐 Demo 叙事

1. 主人给花花戴上 PawRoom Collar。
2. 项圈采集活动、区域、定位和静息趋势。
3. 上班时，PawRoom 桌面端显示“花花正在客厅安睡，状态稳定”。
4. 如果长时间静止、离开安全区或生命状态趋势异常，软件低干扰提醒。
5. 晚上自动生成“花花今日小剧场”，把数据变成可爱的回放。

这条叙事比“我们做了一个健康项圈”更有差异，也更能接住前面已经完成的软件平台。

## 3. v0.3 不改变的约束

- 不新增顶部全局栏。
- 不新增右侧固定抽屉。
- 不把页面改成通用后台 dashboard。
- 不直接把整张高保真图作为页面背景。
- 不像素化文字、按钮和数据卡。
- 不新增信息架构中没有的一级模块。

## 4. v0.3 优先修正

### 4.1 桌面小世界

目标：最接近 `desktop-world-reference.png`。

应保留：

- 左侧液态玻璃导航。
- 大面积像素房间。
- 状态浮层、安全等级、当前区域、项圈连接、电量。
- 桌面小窗预览。
- 安全区虚线、路径虚线、区域标签。
- 摸摸、投喂、叫名字、扔玩具。
- 活动/休息数据小卡。

需要修正：

- 场景底层从“几何 CSS 房间”提升为“像素房间感”：增加木地板、窗、书架、地毯、沙发、玩具、饭盆、宠物窝的像素层次。
- 浮层透明度更接近液态玻璃，降低纯白卡片感。
- 路径和安全区要更像叠在场景上的 HUD，而不是普通线框。

### 4.2 宠物数据

目标：保持数据页结构，不套用首页大橙色 hero。

应保留：

- 顶部状态行。
- 活动量、休息时长、心率、呼吸。
- 家庭区域小地图。
- 数据边界：项圈记录 / 用户补充 / AI 演绎。
- 医疗边界说明。

需要修正：

- 让右侧小地图更像像素家庭区域，而不是普通平面图。
- 数据卡之间的层级更明显：核心数据卡大一些，趋势图更轻。
- 增加“趋势参考，不构成医疗诊断”的显性位置。

### 4.3 今日历程

目标：更像“今日回放创作台”，不是普通时间线页。

应保留：

- 今日小剧场主卡。
- 今日时间线。
- 路径回放。
- 四格漫画、AI 小剧场、可分享导出、Credits 消耗提示。

需要修正：

- 四格漫画占位改成更明显的像素分镜。
- 生成今日回放区域要显示消耗和状态，不只是一颗按钮。
- 时间线播放态要在视觉上更明显，但不夸张。

- 8 个页面仍可访问。

## 1. 品牌体验定位

PawSentinel 的设计系统服务于一个明确的产品承诺：

> 把宠物安全看护从冷冰冰的监控数据，转化为可信、温暖、可长期陪伴的桌面宠物世界。

它不是宠物商城风格，也不是玩具化桌宠风格，更不是医疗监控仪表盘。PawSentinel 的界面需要同时让用户感到三件事：宠物安全可见、数据足够可信、视觉足够柔和。

| 体验层 | 设计表达 |
| --- | --- |
| Safety | 清晰的安全等级、设备状态、连接状态和提醒边界 |
| Presence | 用户自己的宠物分身持续存在于桌面小世界 |
| Calm | 低干扰、低焦虑、适合工作时长期打开 |
| Craft | 像高端消费电子产品，而不是临时 Demo 页面 |
| Delight | 宠物动作、路径、卡片和记忆工坊提供轻量情绪价值 |

## 2. 设计系统定位

PawSentinel 的第一版设计系统服务于一个核心界面：

> 上班中的主人，通过 Web 平台低干扰地看护宠物，并把项圈数据转译成一个温暖、可信、可互动的宠物小世界。

因此 PawSentinel 不是普通宠物商城，也不是纯娱乐桌面宠物分身。它的视觉系统必须同时满足三件事：

1. **安全可信**：项圈数据、位置路径、提醒、生命状态趋势必须清楚。
2. **低干扰陪伴**：界面可以长期停留在电脑上，不刺眼、不抢注意力。
3. **情绪温度**：宠物小世界、今日历程、AI 创作不能像冷冰冰监控面板。

设计关键词：

- Liquid Glass
- Warm Care
- Quiet Dashboard
- Pet Data World
- Soft Spatial UI
- Low Anxiety Monitoring

## 3. Apple-Inspired 原则

PawSentinel 可以借鉴 Apple 设计系统的底层方法，但不要直接复制 Apple 外观。

### 2.1 内容优先

地图、宠物状态、路径、提醒和时间线是内容。左侧任务栏是唯一全局操作层，其他信息都应回到当前页面主画布内部。PawSentinel 不使用顶部全局栏，也不使用右侧固定抽屉，避免横向压缩主画布。

设计要求：

- 主地图/小世界始终是最大视觉区域。
- 左侧任务栏是唯一全局导航，保持窄而稳定，不抢主画布。
- 当前任务相关信息在主画布中以内联卡片、悬浮小窗或底部时间线呈现。
- 玻璃层用于组织信息，不作为纯装饰。

### 2.2 层级清晰

借鉴 Apple 的空间层级：

1. 背景层：温暖环境底色。
2. 内容层：地图、小世界、路径、时间线。
3. 玻璃层：左侧任务栏、页面内浮动控件、底部时间线。
4. 强调层：异常提醒、主操作、当前选中状态。

### 2.3 克制的玻璃感

Liquid Glass 不是“到处透明”。PawSentinel 的玻璃感只用于：

- 左侧任务栏。
- 左侧任务栏内的全局状态区。
- 主画布内的轻量浮层。
- 地图上的悬浮图例。
- 时间线浮层。
- 弹出详情层。

不要用于：

- 大段正文。
- 数据图表背景。
- 异常提醒正文。
- 表单密集区域。

### 2.4 动效服务状态

动效表达状态变化，不做无意义漂浮。

推荐动效：

- 宠物位置从 A 区域移动到 B 区域。
- 安全区边界出现轻微呼吸提示。
- 异常提醒从当前页面内的提醒卡或轻弹层出现。
- 今日历程时间线随播放进度移动。
- 桌面小世界中宠物分身进行低频动作。

避免动效：

- 背景持续大幅流动。
- 卡片无意义漂浮。
- 大面积闪烁。
- 每个按钮都有夸张动效。

| `color.status.safe` | `#68AB12` | 安全、正常、连接成功 |

### 3.2 语义色

| 语义 | Token | 说明 |
| --- | --- | --- |
| 正常 | `color.semantic.success` | 连接正常、安全等级高、状态稳定 |
| 注意 | `color.semantic.warning` | 轻微关注、需要稍后观察 |
| 异常 | `color.semantic.danger` | 离开安全区、持续异常、项圈断连 |
| AI 创作 | `color.semantic.creation` | 今日历程、记忆演绎、表情包 |
| 数据路径 | `color.semantic.path` | 活动路径、历史轨迹、时间线 |

### 3.3 色彩使用比例

推荐比例：

- 70% 近白/奶油白界面底色。
- 15% 浅灰线框、地图线、分割线。
- 10% 暖橙强调。
- 5% 绿色/红色状态色。

禁止：

- 用大面积纯橙铺满 dashboard。
- 用红色表达普通关注状态。
- 只靠颜色表达状态，必须配合文字或图标。
- 用紫蓝渐变表达 AI。

## 5. 字体与文字层级

### 4.1 字体族

Web 推荐：

```css
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text",
  "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

原则：

- 中文优先保证清晰，不追求装饰字体。
- 数字使用 tabular figures，避免图表和时间线跳动。
- 不使用过窄字距。

### 4.2 字号层级

| Token | Size | Line-height | 用途 |
| --- | --- | --- | --- |
| `type.display` | 32 | 40 | 页面主标题，仅少量使用 |
| `type.title.lg` | 24 | 32 | 模块标题、页面标题 |
| `type.title.md` | 20 | 28 | 状态卡标题 |
| `type.body.lg` | 17 | 26 | 关键说明 |
| `type.body.md` | 15 | 24 | 常规正文 |
| `type.body.sm` | 13 | 20 | 辅助信息、标签 |
| `type.caption` | 12 | 18 | 图表标注、时间戳 |

### 4.3 文案风格

PawSentinel 的文案要像“日常看护助手”，不要像医疗设备或监控系统。

推荐：

- “Coco 正在客厅休息，状态稳定。”
- “过去 30 分钟活动偏低，建议稍后观察。”
- “离开安全区 2 分钟，已记录到时间线。”
- “趋势略有波动，不代表医疗诊断。”

避免：

- “系统诊断异常。”
- “疑似患病。”
- “实时监控宠物一举一动。”
- “AI 判断宠物想你了。”

## 6. 布局系统

### 5.1 基准画布

当前 Web 原型基准：

- 设计尺寸：`1440 × 1024`
- 页面类型：桌面 Web dashboard
- 左侧任务栏固定，并承担全局导航与关键全局状态。
- 不设置顶部全局栏。
- 主内容区从左侧任务栏右侧开始展开，根据页面变化。

### 5.2 全局骨架

推荐布局：

| 区域 | 尺寸建议 | 说明 |
| --- | --- | --- |
| 左侧栏 | 112-148px 宽 | 唯一全局栏，只承载 8 个入口 |
| 主画布 | 剩余全部宽度 | 地图、小世界、数据、今日历程、设置与状态信息都在主画布内完成 |
| 页面局部标题区 | 0-56px 高 | 非全局栏，只属于主画布内部，可按页面需要出现 |
| 底部时间线/控制条 | 0-128px 高 | 只在首页、桌面小世界、今日历程等需要时出现 |

不设置顶部全局栏，不设置右侧固定抽屉。

### 5.3 左侧栏

左侧栏是全产品唯一稳定锚点，只包含 8 个点：

1. 首页
2. 桌面小世界
3. 宠物数据
4. 今日历程
5. 设置
6. 新增宠物
7. 电量
8. 宠物信息

入口性质：

| 位置 | 入口 | 性质 | 说明 |
| --- | --- | --- | --- |
| 主任务区 | 首页、桌面小世界、宠物数据、今日历程、设置 | 页面导航 | 对应用户最常进入的 5 个页面 |
| 动作区 | 新增宠物 | 全局动作 | 打开渐进式宠物创建流程 |
| 状态区 | 电量 | 设备状态入口 | 展示项圈电量、连接状态，点击后在主画布内查看详情 |
| 对象区 | 宠物信息 | 当前宠物入口 | 展示当前宠物头像/名称，点击后在主画布内查看档案或切换宠物 |

任务栏原则：

- 只展示图标 + 短标签。
- 当前页使用玻璃胶囊高亮。
- 未读、异常、低电量使用小圆点或小数字，不大面积红色。
- 不额外设置品牌区、顶部全局栏或右侧固定抽屉。
- 异常处理不作为常驻入口，只在首页、宠物数据或电量入口中触发内联提醒/轻弹层。

全局状态迁移规则：

- 当前宠物：放在左侧栏「宠物信息」入口。
- 新增宠物：放在左侧栏「新增宠物」入口。
- 电量：放在左侧栏「电量」入口，并可显示低电量状态点。
- 项圈连接：在首页或宠物数据页主画布内展示，异常时同步到「电量」入口的小状态点。
- 日期、天气：不作为全局栏常驻内容，只在首页或今日历程页面内部出现。
- 通知：不作为顶部铃铛常驻，合并进首页提醒区域或左侧入口小红点。

### 5.4 页面布局模板

#### A. 首页

结构：

- 主画布：家庭地图/活动路径，占据最大面积。
- 主画布上层：一句话状态、安全等级、当前区域、今日概览。
- 主画布下层：最近事件、异常提醒、今日时间线。
- 快捷操作：进入桌面小世界、查看宠物数据、查看今日历程。

用途：

- 默认入口。
- 用户上班时长期停留。
- 异常处理以内联提醒或轻弹层出现，不做右侧栏。

#### B. 桌面小世界

结构：

- 主画布：卡通家庭地图和宠物实时分身。
- 场景内浮层：状态动效、轻互动、跨端状态预览。
- 底部：实时事件或小游戏控制条。

原则：

- 它不是纯小游戏。
- 动作和场景由项圈数据驱动。
- 控制项必须贴近主画布，不形成右侧常驻栏。

#### C. 宠物数据

结构：

- 主画布上方：连接状态、电量、更新时间。
- 主画布中部：活动趋势、家庭区域分布、生命状态趋势。
- 主画布下方：数据边界说明和非诊断提示。

原则：

- 先给结论，再给数据。
- 保留数据边界说明：项圈记录、用户补充、AI 演绎。

#### D. 今日历程

结构：

- 主画布左侧/上方：今日时间轴、路径回放、用户补充事件。
- 主画布右侧/下方：AI 记忆演绎、四格漫画、表情包、角色卡。
- 底部：分享导出和 AI 创作额度提示。

原则：

- 区分项圈记录、用户补充、AI 演绎。
- AI 创作额度只在生成前提示。

#### E. 设置

结构：

- 主画布：安全区域、提醒偏好、AI 创作额度、隐私、医疗边界。
- 设备状态可以作为设置页中的一个分组，但不使用右侧栏。

原则：

- 不做杂项堆叠。
- 设置围绕安全、提醒、额度、隐私组织。

#### F. 新增宠物

结构：

- 从左侧栏「新增宠物」进入。
- 主画布使用渐进式 stepper：选择照片/示例 -> 基础信息 -> 视觉风格 -> 生成分身 -> 连接项圈。
- 每一步只保留一个主要动作和一个预览区域。

原则：

- 渐进式填写。
- 可保存草稿。
- 可以使用示例宠物跳过。

#### G. 电量

结构：

- 从左侧栏「电量」进入。
- 主画布显示项圈电量、连接状态、更新时间、低电量提醒和充电建议。
- 低电量时左侧入口出现小状态点。

原则：

- 电量是安全看护的可信基础。
- 不把电量藏在页面角落，必须作为左侧栏固定状态入口。

#### H. 宠物信息

结构：

- 从左侧栏「宠物信息」进入。
- 主画布显示宠物头像、昵称、品种、年龄、体重、性格标签、声音和照片素材。
- 支持切换宠物或进入宠物档案编辑。

原则：

- 宠物信息是当前对象入口。
- 只展示与“这是我的宠物”相关的信息，不承载通用设置。

## 7. Liquid Glass 材质规范

### 6.1 材质层级

| 材质 | 用途 | 建议参数 |
| --- | --- | --- |
| `glass.taskbar` | 左侧任务全局栏 | 背景 `rgba(255,255,255,0.66)`，blur 24，边框 `rgba(255,255,255,0.72)` |
| `glass.inlinePanel` | 主画布内轻量面板 | 背景 `rgba(255,255,255,0.72)`，blur 20，边界清晰 |
| `glass.floating` | 地图图例、时间浮层 | 背景 `rgba(255,255,255,0.62)`，blur 20 |
| `surface.solid` | 数据卡、表单卡 | 背景 `#FAF8F8`，不透明，保证可读 |

### 6.2 玻璃边界

玻璃层必须有边界，否则会糊成一片。

边界规则：

- 1px 高亮边：顶部/左侧可更亮。
- 1px 暗边：底部/右侧可更暗。
- 阴影轻，不做厚重拟物。
- 玻璃上文字必须达到可读对比。

### 6.3 背景适配

地图、小世界、图表都可能在玻璃层下方。为保证可读性：

- 玻璃层下方先铺一层浅雾化 scrim。
- 重要文字所在区域不要透明过高。
- 图表卡使用 solid surface，不使用玻璃。

## 8. 圆角、间距、描边、阴影

### 7.1 圆角

| Token | Radius | 用途 |
| --- | --- | --- |

| `radius.pill` | 999 | 胶囊标签、当前选中态 |

### 7.2 间距

使用 4/8pt 系统：

```text
4, 8, 12, 16, 20, 24, 32, 40, 48, 64
```

页面建议：

- 页面外边距：16-24。
- 卡片间距：16。
- 大区块间距：24。
- 主画布内面板间距：12-16。
- 左侧任务栏图标项高度：56-64。

### 7.3 描边

| Token | Value | 用途 |
| --- | --- | --- |
| `stroke.hairline` | 1px | 玻璃边界、分隔线 |
| `stroke.default` | 1.5px | 卡片边界 |
| `stroke.emphasis` | 2px | 当前选中、安全区、关键路径 |

### 7.4 阴影

| Token | 用途 | 建议 |
| --- | --- | --- |
| `shadow.soft` | 普通卡片 | 低透明、低扩散 |
| `shadow.float` | 悬浮图例/浮层 | 中等扩散 |
| `shadow.panel` | 主画布内浮层/面板 | 轻微深度，不抢主内容 |

阴影不要变成电商卡片式厚重投影。PawSentinel 是工作台，不是商品展示页。

## 9. 数据可视化规范

PawSentinel 的图表不是分析后台，而是日常看护状态解释。

### 8.1 图表类型

| 数据 | 图表 |
| --- | --- |
| 活动量 | 24h 柱状图 |
| 休息时长 | 分段柱状/进度条 |
| 心率趋势 | 细折线图 |
| 呼吸趋势 | 细折线图 |
| 区域停留 | 横向条形图 |
| 今日路径 | 地图路径线 |
| 今日历程 | 时间轴 + 路径同步 |

### 8.2 图表原则

- 先显示结论，再显示细节。
- 图表不使用强饱和大面积色。
- 异常点用点位和文字提示，不只用红色。
- 所有趋势必须标注“趋势参考 / 非诊断”边界。
- 时间轴应使用明确时间刻度，不让用户猜。

## 10. 状态系统

### 9.1 安全状态

| 状态 | 文案 | 视觉 |
| --- | --- | --- |
| 正常 | 状态稳定 | 绿色小标记 + 温和说明 |
| 轻微关注 | 建议稍后观察 | 金色/橙色提示 |
| 需查看 | 建议查看原因 | 橙红提示，但不恐吓 |
| 断连 | 项圈未连接 | 灰色 + 明确恢复操作 |

### 9.2 数据来源状态

| 来源 | 标记 |
| --- | --- |
| 项圈记录 | `Device Record` |
| 用户补充 | `Owner Note` |
| AI 演绎 | `AI Story` |

AI 演绎不得直接显示为事实。

## 11. 图标系统

暂不设计具体按钮，但需要确定图标风格。

图标原则：

- 使用线性图标。
- 线宽 1.75-2px。
- 圆角端点。
- 不使用 emoji 作为功能图标。
- 图标必须配文字标签，至少在首版高保真中配短标签。

推荐后续图标方向：

- 项圈：简化项圈轮廓。
- 地图：区域线框。
- 小世界：房间/窗口。
- 回放：时间线/播放。
- 设置：标准齿轮。
- 提醒：铃铛/提示点。

## 12. 表单与渐进式创建

新增宠物流程必须使用渐进披露。

步骤：

1. 上传素材。
2. 填写必要信息。
3. 选择视觉风格。
4. 生成分身。
5. 连接项圈。
6. 进入首页。

首屏只要求：

- 宠物照片或示例宠物。
- 宠物名。
- 类型。

可稍后补充：

- 体重。
- 性格标签。
- 声音。
- 更多照片。
- 品种细节。

原则：

- 每步一个主任务。
- 可保存草稿。
- 可以使用示例宠物跳过。
- 不用一次性填完全部信息。

## 13. 响应式规则

第一版重点是桌面 Web，但要预留适配。

### 12.1 Desktop 1440+

- 左侧任务栏固定，作为唯一全局栏。
- 不设置右侧固定抽屉，主画布占据左侧栏之外的全部宽度。
- 地图/小世界作为主画布。
- 不出现顶部全局栏。

### 12.2 Tablet 1024-1439

- 左侧任务栏压缩为图标与关键状态点。
- 不设置右侧固定抽屉，页面信息在主画布内重排。
- 底部时间线保留。

### 12.3 Mobile <768

- 不建议完整承载 Web 工作台。
- 手机端只做状态预览、提醒、回放摘要。
- 桌面小世界在手机上降级为卡片式状态流。

## 14. 可访问性规则

必须满足：

- 正文对比度至少 4.5:1。
- 重要状态不能只靠颜色。
- 点击目标不小于 44px。
- 键盘焦点可见。
- 图表提供文字摘要。
- 动效支持 reduced-motion。
- 异常提醒可被关闭、稍后提醒、查看原因。

玻璃拟态尤其要注意对比度。模糊和透明不是牺牲可读性的理由。

## 15. 边界说明

本 v0.1 暂不定义：

- 具体按钮组件样式。
- 宠物 IP 形象细则已拆到独立组件库文档。
- 高保真插画风格。
- 真实 3D/Live2D/像素角色规范。
- 品牌 logo 最终形态。
- 完整移动端设计系统。

这些应在低保真页面确认后进入 v0.2。

## 16. 高保真执行顺序

建议高保真推进顺序：

1. 首页：确立地图、玻璃层、主画布状态卡、左侧单一全局任务栏。
2. 桌面小世界：确立宠物分身、小世界画布、互动层。
3. 宠物数据：确立图表、趋势、非诊断提示。
4. 今日历程：确立 AI 创作和事实/演绎边界。
5. 新增宠物：确立渐进式 onboarding。
6. 设置：确立隐私、额度、提醒设置。

第一张高保真建议从“首页”开始，因为它能同时验证色彩、玻璃材质、任务栏、地图、数据卡和状态系统。

# PawRoom 高保真页面视觉审查 v0.9

日期：2026-07-08  
背景：首页已确定采用 v0.8 睡猫床垫方向：橙色玻璃上半区、像素猫单体视觉、下半数据卡片、左侧栏固定。

---

## 1. 总结结论

其他高保真页面需要优化。

原因不是功能不对，而是当前除首页外的页面大多来自 v0.3 阶段，视觉语言仍然是“像素场景 + 液态玻璃卡片”的旧方向。现在首页已经升级为更强的“橙色玻璃主视觉 + 大胆数据卡 + 单体宠物情绪视觉”，如果其他页面不跟进，会出现明显风格断层。

---

## 2. 页面优化优先级

| 页面 | 是否建议优化 | 优先级 | 原因 |
| --- | --- | --- | --- |
| 桌面小世界 | 是 | P0 | 核心体验页，必须承接首页“进入桌面小世界”的期待。 |
| 宠物数据 | 是 | P0 | 首页下半部分已经建立了新的数据卡风格，数据页必须同步。 |
| 今日历程 | 是 | P1 | 是 AI 记忆与内容创作的关键差异化页面，需要更强情绪化表达。 |
| 添加宠物 | 是 | P1 | 当前应改为渐进式创建体验，并匹配新的 IP/橙色玻璃风格。 |
| 宠物信息 | 是 | P2 | 可以优化为“宠物身份 + 素材资产库”，但不如 P0/P1 急。 |
| 电量 | 是 | P2 | 可优化为设备状态页，视觉跟随首页即可。 |
| 设置 | 轻度优化 | P2/P3 | 功能型页面不需要太花，但需要统一色彩、卡片和左侧栏。 |

---

## 3. 各页面优化方向

### 3.1 桌面小世界

建议保留“像素小世界”，但不再像旧版那样铺满一个普通房间地图。

新方向：

- 上半部分做“当前互动场景”大视觉；
- 主场景可以是像素猫在床垫、玩具、窗边、饭盆等状态间切换；
- 橙色玻璃面板承载状态与操作；
- 下方放互动按钮：摸摸、投喂、叫名字、扔玩具；
- 右下或底部放小组件预览、手机灵动岛预览。

### 3.2 宠物数据

需要跟首页的数据卡保持同一语言。

新方向：

- 大橙色主卡展示今日总览；
- 活动、休息、心率、呼吸、区域停留做错位数据磁贴；
- 使用橙、绿、蓝、紫分组；
- 避免一整页同质白卡；
- 明确“趋势参考，不是医疗诊断”。

### 3.3 今日历程

这一页应更像“宠物一天的故事编辑台”，而不是普通时间线。

新方向：

- 上方橙色玻璃卡展示“今日小剧场”；
- 中间为时间线：09:05、09:40、10:42 等；
- 右侧或下方展示四格漫画、表情包、角色卡预览；
- 保留 AI 演绎 / 用户补充 / 项圈记录的边界说明；
- CTA：生成今日回放、分享导出。

### 3.4 添加宠物

需要从“表单页”改为“渐进式创建流程”。

新方向：

- Step 1 上传照片 / 选择示例宠物；
- Step 2 基础信息；
- Step 3 选择像素风格；
- Step 4 生成宠物分身；
- Step 5 连接项圈；
- 每一步只显示必要信息，减少负担。

### 3.5 宠物信息

建议变成“宠物档案 + 素材库”。

新方向：

- 大头像 / 睡猫 IP 卡；
- 基础信息：年龄、品种、体重、性格；
- 素材库：照片、声音、表情包、今日事件；
- 设备绑定信息；
- CTA：补充素材、切换宠物。

### 3.6 电量

建议作为设备状态中心。

新方向：

- 大号电量 78%；
- 项圈在线、预计可用时长、同步时间；
- 低电量提醒设置；
- 同步频率；
- 橙色玻璃主卡 + 少量白色数据卡。

### 3.7 设置

轻度统一即可。

新方向：

- 安全区域；
- 提醒偏好；
- AI 创作额度；
- 数据隐私；
- 医疗边界说明；
- 使用更清晰的分组卡片和开关控件，不需要过强主视觉。

---

## 4. 建议执行顺序

1. 先优化桌面小世界；
2. 再优化宠物数据；
3. 再优化今日历程；
4. 然后补添加宠物；
5. 最后统一宠物信息、电量、设置。

如果时间有限，至少先完成前三个：桌面小世界、宠物数据、今日历程。

---

## 5. 统一视觉约束

后续所有页面应继承首页 v0.8 的视觉基准：

- 左侧栏保持不变；
- 橙色玻璃作为核心主视觉；
- 像素宠物作为情绪锚点；
- 数据卡采用大胆色块和错位磁贴；
- 不使用顶部全局栏；
- 不使用右侧抽屉栏；
- 文本、图表、按钮不能像素化；
- 页面尺寸保持 `1440 × 1024`。

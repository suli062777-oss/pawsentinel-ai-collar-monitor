# PawRoom image2 透明素材拆分清单 v0.1

本清单用于把现有高保真图还原为前端时，区分“前端可绘制结构”和“需要 image2 单独生成/修复的透明素材组件”。目标不是重新设计页面，而是提升对设计稿中像素宠物、像素场景和装饰资产的还原度。

## 使用原则

- UI 容器、玻璃卡片、按钮、数据图表、路径线、安全区、文字层级优先用前端实现，便于后续接真实数据。
- 宠物 IP、床垫、室内像素场景、漫画格、桌面小窗预览、项圈设备等视觉资产优先拆成透明 PNG/WebP。
- image2 输出物不带文字、不带 UI 卡片、不带复杂阴影，阴影和玻璃叠层由前端控制。
- 所有像素风资产保持统一视角、统一光源、统一橙色项圈识别点。
- 输出后统一放入 `frontend/assets/generated/`，再由组件引用，避免把图片直接写死进页面结构。

## image2 通用提示词模板

```text
生成一个 PawRoom 产品用的透明底 2D 像素风素材组件：
对象：[具体素材名称]
风格：温暖、可爱、精致的 2D pixel art，统一暖橙色识别点 #FF8F3F，宠物佩戴橙色智能项圈
视角：[正面 / 轻俯视 / 等距俯视]
要求：transparent background, isolated object, no text, no UI card, no watermark, crisp pixel edges, soft warm lighting, clean silhouette
输出：PNG/WebP，长边至少 1024px，边缘干净，适合叠加在液态玻璃 UI 上
```

## 优先级

1. `P0`：桌面小世界背景、宠物待机分身、首页睡觉宠物、四格漫画面板。
2. `P1`：饭盆/玩具/宠物窝/沙发等房间道具、床垫透明图、项圈设备图。
3. `P2`：素材库小图标、桌面小窗缩略图、AI 小剧场封面。

## 验收标准

- 前端截图中不出现“被裁切的整张高保真图直接贴上去”的痕迹。
- 宠物和场景是像素风，UI 控件仍保持液态玻璃和 #FF8F3F 橙色体系。
- 透明素材不承载功能文字，功能状态仍由前端组件渲染。
- 后续接真实数据时，只替换数据源和素材 URL，不需要重排页面。

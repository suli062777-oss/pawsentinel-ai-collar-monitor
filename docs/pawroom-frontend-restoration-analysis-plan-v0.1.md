# PawRoom 前端高保真还原分析与执行计划 v0.1

日期：2026-07-08  
目标：将现有高保真图片还原为可交互前端页面，优先最大化还原设计稿的结构、比例、层级、留白、圆角、配色和视觉氛围，而不是重新优化设计。

- 当前清点图片：8 张 PNG，原始尺寸均约为 `1487 x 1058`，比例与 `1440 x 1024` 基准画布接近。
- 主框架总览图：`docs/assets/pawroom-main-framework-pages-contact-sheet.png`

## 1. 还原总原则

1. 先按图片拆结构，再实现组件。
2. 不自行改变布局、风格、配色、圆角、留白。
3. 不擅自增加设计稿中没有的模块。
4. 不把页面改成通用后台 dashboard。
5. 优先还原页面结构、比例、层级、留白、视觉氛围。
6. 资源缺失时允许占位，但位置、尺寸、层级必须一致。
7. 未来要接真实功能的区域必须组件化和数据化，不能把业务逻辑写死。
8. 图片中的中文 UI 文案要用真实 DOM 文本重建，避免直接依赖 AI 图中的模糊文字。
9. 像素化只作用于宠物、房间、地图、小剧场、表情包等内容素材；UI 框架、按钮、卡片、图表和文字保持现代液态玻璃风格。

## 2. 页面结构拆解

### 2.1 全局 App Shell

- 画布：`1440 x 1024` 作为设计基准。
- 背景：温暖奶油白或极浅暖灰。
- 左侧栏：唯一全局导航，不设置顶部全局栏，不设置右侧固定抽屉。
- 左侧栏上半区：`首页 / 桌面小世界 / 宠物数据 / 今日历程 / 设置`。
- 左侧栏下半区：`添加 / 电量 / 宠物信息`。
- 主内容区：从左侧栏右侧开始，根据当前页面切换布局。

### 2.2 首页

- 左侧：固定液态玻璃导航栏。
- 上半主视觉：橙色玻璃大面板，展示当前状态、关键大数字、像素宠物单体视觉。
- 中间：状态时间点或轻量时间线，连接上方状态和下方数据。
- 下半：错位数据卡，包括活动量、休息时长、心率趋势、呼吸趋势、提醒、数据置信度。
- 重点：保留已选定首页风格，不回退到地图型首页。

### 2.3 桌面小世界

- 主体：沉浸式 2D 像素房间/家庭场景，占据主画布最大面积。
- 浮层：状态卡、安全等级、当前区域、项圈连接、电量、桌面小窗预览。
- 交互区：摸摸、投喂、叫名字、扔玩具等按钮贴近场景。
- 路径/区域：虚线安全区、活动路径、区域标签。
- 底部/角落：活动量、休息时长、AI 演绎说明等轻量数据卡。

### 2.4 宠物数据

- 顶部/上区：连接状态、电量、最近同步、数据置信度、当前区域。
- 中部：活动量、休息时长、心率趋势、呼吸趋势、区域停留、活动节律等数据磁贴。
- 辅助：小型像素家庭区域图只作为数据卡，不作为全页主体场景。
- 底部：项圈记录、用户补充、AI 演绎边界，以及“趋势参考，不构成医疗诊断”。

### 2.5 今日历程

- 顶部：今日小剧场/今日摘要主卡。
- 中部：时间线与路径回放，保留具体时间节点。
- 创作区：AI 小剧场、四格漫画、表情包、角色卡、短动画预览。
- 底部：Credits 消耗提示、生成今日回放、分享导出。
- 重点：它是“故事编辑台”，不是普通后台时间线。

### 2.6 设置

- 主体：安全区域、提醒偏好、Paw Credits、数据隐私、医疗边界说明。
- 表现：功能型分组卡片，不做过强主视觉。
- 控件：开关、单选、滑块、说明卡、危险操作区。

### 2.7 添加宠物

- 使用渐进式 Stepper。
- Step 1：上传照片或选择示例宠物。
- Step 2：填写必要基础信息。
- Step 3：选择视觉风格。
- Step 4：生成宠物分身预览。
- Step 5：连接项圈。
- 每一步只保留一个主要动作和一个预览区，降低填写负担。

### 2.8 电量

- 主卡：大号电量百分比、项圈在线状态、预计可用时长、最近同步。
- 辅助卡：低电量提醒、同步频率、充电建议、设备信息。
- 低电量状态同步到左侧栏电量入口的小状态点。

### 2.9 宠物信息

- 主体：宠物头像/像素 IP、昵称、品种、年龄、体重、性格标签。
- 素材区：照片、声音、表情包、今日事件、宠物角色卡。
- 设备区：当前绑定项圈与同步状态。
- 操作：补充素材、切换宠物、编辑档案。

## 4. 布局关系

- 使用设计基准变量：`--design-width: 1440`，`--design-height: 1024`。
- 第一版优先做桌面基准还原，主舞台可按比例缩放，但内部比例以 `1440 x 1024` 为准。
- 左侧栏宽度按设计稿控制在 `112-148px` 区间，不随内容撑开。
- 主画布使用 page-level CSS Grid 或绝对定位混合方式，先保证设计稿位置准确。
- 所有页面共享左侧栏，主内容只替换当前页面内部结构。
- 页面之间的差异应来自主画布结构，不来自全局框架变化。
- 场景型页面优先使用背景素材 + DOM 浮层；数据型页面优先使用 DOM 卡片 + SVG 图表。

## 5. 字体层级

采用设计系统字体族：

```css
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text",
  "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

层级：

- Display：32/40，用于首页大标题或主状态。
- Title LG：24/32，用于页面标题、模块标题。
- Title MD：20/28，用于状态卡标题。
- Body LG：17/26，用于关键说明。
- Body MD：15/24，用于常规文案。
- Body SM：13/20，用于标签和辅助信息。
- Caption：12/18，用于图表刻度、时间戳。
- 数字使用 `font-variant-numeric: tabular-nums`，避免指标跳动。

## 6. 色彩层级

当前主色按用户要求以 `#FF8F3F` 为基准。与设计系统合并为：

- 主强调：`#FF8F3F`，用于当前页面、主 CTA、路径点、关键数据主卡。
- 页面底色：`#FAF8F8` / `#F6F1E8`，用于奶油白背景和次级背景。
- 安全状态：`#68AB12`，用于正常、连接成功、安全等级高。
- 注意状态：暖橙/金色，用于轻微关注。
- 危险状态：`#E2574C`，只用于真正异常，不大面积使用。
- 数据路径：`#7DA4BD` 或柔和蓝，用于历史路径、呼吸、辅助线。
- 主文字：`#2B2521`。
- 二级文字：`#4D4741`。
- 辅助文字：`#8D8B73`。
- 描边：`#D9D2C8` 或白色半透明边界。

## 7. 需要精确还原的关键点

1. 左侧栏必须保持上 5 + 下 3 的分组位置。
2. 不出现顶部全局栏。
3. 不出现右侧固定抽屉。
4. 页面尺寸和比例以 `1440 x 1024` 为准。
5. 首页保持已选定的橙色玻璃睡猫方向。
6. 桌面小世界保持沉浸式像素房间和玻璃浮层，不改成数据后台。
7. 宠物数据保持数据仪表盘，不改成首页结构。
8. 今日历程保持时间线 + 路径回放 + AI 创作输出。
9. 文案必须使用真实中文 DOM，不直接依赖图片里的模糊字。
10. 图标不能用 emoji，使用线性 SVG 图标。
11. 像素宠物和场景素材缺失时，先用占位保持结构；后续用 image2 生成透明底高清组件。
12. 医疗边界必须保留：趋势参考，不构成医疗诊断。

## 8. 还原风险点

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| 高保真图文件名无语义 | 容易页面对应错误 | 先生成总览图并建立页面-文件映射表 |
| AI 图中文字模糊或乱码 | 前端直接贴图会不专业 | 所有 UI 文案重建为 DOM 文本 |
| 场景细节过复杂 | 纯 CSS 难以复刻 | 场景用图片素材承载，交互层用 DOM/SVG 覆盖 |
| 透明素材缺失 | 宠物/床垫/项圈边缘不干净 | 用 image2 重新生成透明底高清组件 |
| 源图尺寸不是 1440x1024 | 还原比例可能偏移 | 实现统一使用 1440x1024 设计坐标系 |
| 过度组件化导致位置漂移 | 设计稿还原度下降 | 第一轮组件化只抽稳定结构，页面局部允许精确定位 |
| 图表和数据卡泛后台化 | 风格偏离 C 端陪伴产品 | 按设计稿使用错位磁贴、玻璃层和温暖色块 |
| 中文字体渲染差异 | 文案换行和卡片高度变化 | 锁定字体栈、行高、字重和 tabular numbers |
| 动效过多 | 偏离低干扰看护 | 动效只服务状态变化，避免大面积装饰动画 |

## 10. 每次大更新固定审查流程

1. 运行页面。
2. 截取 `1440 x 1024` 页面截图。
3. 与原始高保真图并排检查。
4. 检查中文是否乱码。
5. 检查文本是否溢出或遮挡。
6. 检查左侧栏位置和当前态是否一致。
7. 检查是否误加顶部栏或右侧栏。
8. 检查颜色是否偏离 `#FF8F3F` 主色和奶油白底色。
9. 检查像素素材是否只出现在内容区，而不是污染 UI 控件。
10. 记录问题，完成二次修正后再进入下一阶段。

## 2. 低保真设计定位

这版不是高保真界面，也不是最终 UI。它的目标是先确定：

- Web 平台左侧栏 8 个入口如何组织。
- 每个入口进入后，主画布的功能区怎么摆。
- 用户从新增宠物到日常看护，再到桌面小世界和今日历程的闭环怎么走。
- 异常提醒出现后，用户在首页、宠物数据或电量入口中如何查看原因和建议操作。

视觉上延续参考库的底层结构特征：

- 暖色背景。
- 卡片式模块。
- 圆角工作台布局。
- 左侧栏 + 主画布结构。

硬性结构规则：

- 页面尺寸统一为 `1440 × 1024`。
- 只保留左侧栏，不设置顶部全局栏。
- 只保留左侧栏，不设置右侧抽屉栏或右侧固定状态栏。
- 状态、提醒、数据、操作都放进当前页面主画布内，以卡片、浮层或底部控制条呈现。

当前阶段只保留低保真线框，不进入图片、插画、动效、颜色细节和真实 UI 文案。

## 3. 左侧栏信息结构

左侧栏只包含 8 个点：

1. 首页
2. 桌面小世界
3. 宠物数据
4. 今日历程
5. 设置
6. 新增宠物
7. 电量
8. 宠物信息

入口分组建议：

| 分组 | 入口 | 作用 |
| --- | --- | --- |
| 主任务入口 | 首页、桌面小世界、宠物数据、今日历程、设置 | 用户日常切换的 5 个页面 |
| 全局动作 | 新增宠物 | 打开渐进式宠物创建流程 |
| 设备状态 | 电量 | 查看项圈电量、连接状态、低电量提醒 |
| 当前对象 | 宠物信息 | 查看当前宠物档案或切换宠物 |

## 4. 页面清单

### 01 首页

目的：作为 Web 平台默认入口，让用户快速判断宠物是否安全。

页面结构：

- 左侧栏：8 个固定入口。
- 主画布中心：家庭地图、当前区域、活动路径。
- 主画布上方或浮层：一句话状态、安全等级、连接状态。
- 主画布下方：今日概览、最近事件、异常提醒、时间线。
- 快捷操作：进入桌面小世界、查看宠物数据、查看今日历程。

### 02 桌面小世界

目的：展示 PawRoom 的核心差异化，让用户在工作时看到卡通版宠物正在生活。

页面结构：

- 左侧栏：保持不变。
- 主画布中心：卡通家庭地图和宠物实时分身。
- 主画布浮层：状态动效、轻互动、跨端状态预览。
- 底部控制条：实时事件、轻游戏入口、桌面小窗预览。

### 03 宠物数据

目的：建立可信度，让用户知道小世界和提醒由项圈数据驱动。

页面结构：

- 左侧栏：保持不变。
- 主画布上方：连接状态、电量、更新时间。
- 主画布中部：活动量、家庭区域、心率/呼吸/静息趋势。
- 主画布下方：数据边界说明。
- 数据边界必须包含：项圈记录、用户补充、AI 演绎。

### 04 今日历程

目的：把一天的数据转化为情绪价值和可分享内容。

页面结构：

- 左侧栏：保持不变。
- 主画布主区域：今日时间轴、路径回放、用户补充事件。
- 主画布创作区：AI 小剧场、四格漫画、表情包、角色卡。
- 底部：分享导出和 AI 创作额度提示。

### 05 设置

目的：承接配置、隐私和商业化，不抢主线入口。

页面结构：

- 左侧栏：保持不变。
- 主画布：安全区域、提醒偏好、AI 创作额度、数据隐私、医疗边界说明。
- 设备相关设置作为页面内分组，不做右侧固定栏。

### 06 新增宠物

目的：完成“这是我的宠物”的个体化入口。

页面结构：

- 左侧栏：保持不变，高亮「新增宠物」。
- 主画布：渐进式创建流程。
- 流程步骤：选择照片/示例 -> 填写基础信息 -> 选择视觉风格 -> 生成宠物分身 -> 连接项圈。
- 每一步只保留一个主动作和一个预览区。

### 07 电量

目的：让用户快速确认项圈是否在线、是否有足够电量支撑看护。

页面结构：

- 左侧栏：保持不变，高亮「电量」。
- 主画布：项圈电量、连接状态、更新时间、低电量提醒、充电建议。
- 低电量时，左侧栏「电量」入口出现小状态点。

### 08 宠物信息

目的：管理当前宠物身份和素材，保证 AI 分身和今日历程有稳定对象。

页面结构：

- 左侧栏：保持不变，高亮「宠物信息」。
- 主画布：宠物头像、昵称、品种、年龄、体重、性格标签、声音和照片素材。
- 支持切换宠物或进入宠物档案编辑。

## 5. 流程关系

### 5.1 主流程

新增宠物 -> 首页 -> 桌面小世界 -> 今日历程。

这条路径用于演示完整价值闭环。

### 5.2 数据查看分支

首页 -> 宠物数据 -> 电量。

这条路径用于解释状态和提醒的可信来源。

### 5.3 异常流程

首页/宠物数据/电量 -> 页面内异常提醒 -> 设置。

异常处理不是独立侧栏入口，也不是右侧栏，而是当前页面内的提醒卡、轻弹层或状态点。

---

## 6. 设计原则

1. 首页是默认入口，不是桌宠页面。
2. 桌面小世界必须服务“远程看护”和“低干扰陪伴”，不能变成纯小游戏。
3. 宠物数据、异常原因、非诊断提示必须可见。
4. AI 创作能力放在今日历程中，不抢安全看护主线。
5. Paw Credits 在低保真阶段统一表达为“AI 创作额度”，降低理解成本。
6. 所有页面只使用左侧栏 + 主画布，不再使用顶部全局栏和右侧抽屉栏。

---

## 7. 后续高保真推进建议

建议下一步按这个顺序进入高保真：

1. 先做首页，因为它决定产品第一印象和安全心智。
2. 再做桌面小世界，因为它决定产品差异化。
3. 再做宠物数据，因为它决定可信度。
4. 再做新增宠物，因为它决定首次使用门槛。
5. 最后做今日历程、设置、电量和宠物信息。

当前低保真结构已经收敛为“左侧栏 8 入口 + 主画布”的页面拆分方式。
